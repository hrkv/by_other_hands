By Other Hands — прототип
=========================

**Жанр**: онлайн пошаговая тактическая 2D‑игра для двух игроков.  
**Цель этапа A**: подготовить базовую структуру проекта и договориться о соглашениях.

## Структура проекта (черновик)

- `client/` — клиентская часть (рендер поля, UI, ввод игрока, работа с сетью).
- `server/` — серверная часть (матчмейкинг, управление сессиями, валидация ходов).
- `shared/` — общие структуры данных и правила игры, разделяемые клиентом и сервером.
- `assets/` — игровые ассеты (спрайты, звуки, карты, данные уровней).
- `docs/` — дополнительная документация (дизайн‑заметки, схемы и т.п.).
- `PLAN.md` — основной план разработки (этот файл лучше хранить в корне).

## Версионирование

Используется упрощённый SemVer: `0.MAJOR.MINOR` до большого релиза (см. `PLAN.md`).  
Патч‑версии добавятся позже, когда появится публичный релиз.

## Запуск

На этапе A структура и зависимости считаются черновыми, поэтому команды запуска пока не настроены.

План на ближайшие шаги:

1. Выбор конкретного стека (движок/фреймворк) для Web‑клиента.
2. Настройка сборки и dev‑сервера для `client/`.
3. Базовый сервер (`server/`) для матчмейкинга и синхронизации ходов.
4. Общий модуль правил (`shared/`) для единого источника правды по логике игры.

Актуальный дизайн и дорожная карта описаны в `PLAN.md`.
